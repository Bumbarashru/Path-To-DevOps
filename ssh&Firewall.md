#Firewall - фильтр, который проверяет каждый сетевой пакет или запрос приложения
Виды:
    *На базе сети или на базе хоста
    *Аппаратный или программный
    *Фильтрация сетевого или прикладного уровня

Часто встречаются: UFW, Firewalld

UFW:
ufw <опции> <действие> <параметры>
пример:
sudo ufw default deny incoming - запретить всё входящее
sudo ufw default allow outgoing - исходящее разрешаем
sudo ufw allod OpenSSH / sudo ufw allow 22/tcp

список действий:
enable          включить фаерволл и добавить его в автозагрузку
disable         отключить фаерволл и удалить из автозагрузки
reload          перезагрузка
default         задать политику по умолчанию
logging         Включить логирование или изменить уровень подробности
reset           сбросить все настройки для состояние по умолч
status          статус фаерволла
show            посмотреть 1 из отчётов о работе
allow           разрешающее правило
deny            запрещающее правило
reject          отбрасывающее правило
limit           лимитирующее правило
delete          удалить правило 
insert          вставить правило

Firewalld 
firewalld-cmd <опции> <зона> <ПРАВИЛО>
пример открыть порт 22:
firewalld-cmd --permanent--add-port=22/tcp
firewalld-cmd --permanent--zone=external--add-port=22/tcp
firewalld-cmd --permanent--add-port=ssh
несколько предустрановленных зон:
drop - блокировать все входящие пакеты, разрешить только исходящие
block - в отличие от предыдущего варианта отправителю пакета будет отправлено сообщение по блокировке его пакета
public - поддерживать входищие соединения только для ssh, dhclient
external - поддержка NAT для скрытия внутренней сети
internal - разреш ssh, samba, mdns, dhcp
dmz      - используется для изолированных серверов, у которых нет доступа к сети, разрешено только подкл по ssh
work     - разреш ssh, dhcp
home     - аналог internal
trusted  - всё разрешено
