# IaC (Инфраструктура как код)
Простыми словами, это философия управления серверами, сетями, базами данных и балансировщиками через файлы с кодом, а не через нажатие кнопок в веб-интерфейсе (консоли AWS, панель хостинга) или через ручные команды в терминале.

 ##Три золотых правила IaC:

    1) Идемпотентность: Вы можете запустить свой код 1 раз или 100 раз — результат всегда будет один. Если сервер уже настроен, код ничего не сломает.

    2) Версионирование: Ваша инфраструктура хранится в Git (рядом с кодом приложения). Вы можете видеть, кто, когда и зачем изменил конфигурацию базы данных (и откатить, если что).

    3) Самообслуживание: Разработчик может поднять себе полную копию продакшена для тестов, просто запустив скрипт на своем ноутбуке, не дергая админа.

 ##Главные инструменты IaC:

    Terraform (от HashiCorp) — создает сами ресурсы (виртуалки, кластеры Kubernetes, базы данных, сети). Это называется "Декларативный подход" — вы пишете какой результат хотите, а Terraform сам понимает, как к нему прийти.

    Ansible / Puppet / Chef — настраивают конфигурацию внутри уже существующих серверов (устанавливают софт, копируют файлы).

    Kubernetes манифесты (YAML) — описывают, сколько подов с вашим приложением должно работать и куда их маршрутизировать
